Supabase — Banco de Dados Não configurado

Configure o Supabase para histórico compartilhado entre vendedores, autenticação segura e visibilidade do gestor.
Passo a passo abaixo.

Setup em 3 passos
1 Acesse supabase.com → New Project → anote a URL e a anon key
2 No Supabase → SQL Editor → execute o script abaixo para criar as tabelas
3 Cole a URL e a anon key nos campos abaixo e clique em Salvar e Testar
SQL — cole no Supabase SQL Editor
-- Tabela de propostas (histórico compartilhado)
CREATE TABLE IF NOT EXISTS propostas (
  id            BIGSERIAL PRIMARY KEY,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  vendedor_id   INTEGER,
  vendedor_nome TEXT,
  tipo_proposta TEXT DEFAULT 'novo',
  nome_empresa  TEXT,
  crm_cliente   TEXT,
  contato_nome  TEXT,
  contato_email TEXT,
  pacote_horas  TEXT,
  preco_mensalidade TEXT,
  preco_setup   TEXT,
  desconto_pct  INTEGER DEFAULT 0,
  integ_tipo    TEXT,
  whatsapp_info TEXT,
  status_proposta TEXT DEFAULT 'enviada',
  motivo_perda  TEXT,
  payload_json  JSONB,
  data_proposta TEXT,
  validade_proposta TEXT
);

-- Habilita RLS (Row Level Security)
ALTER TABLE propostas ENABLE ROW LEVEL SECURITY;

-- Policy: todos os usuários autenticados podem ler e inserir
CREATE POLICY "Acesso total para autenticados"
  ON propostas FOR ALL
  USING (auth.role() = 'authenticated')
  WITH CHECK (auth.role() = 'authenticated');
